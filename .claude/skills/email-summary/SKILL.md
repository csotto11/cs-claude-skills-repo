# CS - Daily Email Summary

## Goal
Reduce inbox noise by delivering a single daily email that summarizes all emails from the previous day, flags critical ones requiring attention, and lists today's meetings. Non-critical emails are automatically marked as read after the summary is sent.

## Email Sections

### 1. All Emails Summary
A high-level paragraph (3-5 sentences) summarizing everything received the previous day — newsletters, marketing, store promotions, transactional emails, personal messages, everything. The goal is a quick "here's what came in" snapshot so the user never needs to scroll through junk.

### 2. Critical Emails
A bulleted list of emails that require closer attention. An email is "critical" if it meets **any** of these criteria:
- **Personalized** — addressed specifically to the user, not part of a mass distribution or mailing list
- **Financial implications** — bills, payment confirmations, bank alerts, investment notifications, tax documents
- **Action required** — RSVPs, signature requests, approval requests, deadlines, direct questions expecting a reply
- **Time-sensitive** — expiring offers relevant to active decisions, shipping/delivery updates for pending orders

Each bullet should include:
- **From** (sender name)
- **Subject** line
- **1-sentence summary** of what it's about and what action (if any) is needed

### 3. Key Meetings Today
A bulleted list of any meetings/events on the calendar for **today** (the day the summary is sent, not the previous day). Include:
- **Time** (in ET)
- **Title**
- **Location / video link** (if available)

Note: The user doesn't have many meetings on personal email, so this section will often be empty. When empty, display "No meetings scheduled for today."

## Tone & Style
- **Clean and scannable** — no fluff, no filler. This is a utility email, not a newsletter.
- **Professional but casual** — like a briefing from a competent assistant.
- **Mobile-first** — designed to be read on a phone in under 2 minutes.

## Email Design
- **Mobile-first HTML** — single column, max-width 600px.
- Section headers with distinct colors:
  - All Emails Summary: #2c3e50 (dark slate)
  - Critical Emails: #c0392b (red — attention)
  - Key Meetings: #2980b9 (blue)
- Bold sender names and subjects in the critical section for scannability.
- Footer with date and "Generated by Claude" tagline.
- Use the HTML template at `scripts/email_template.html`.

## Post-Send Actions
After the summary email is successfully sent:
1. **Mark non-critical emails as read** — All emails from the previous day that were NOT classified as "critical" should be marked as read in Gmail. This keeps the inbox clean so only actionable items remain unread.
2. Critical emails remain **unread** so they're visible when the user opens Gmail.

## Claude Dispatch Integration

- **Adjust send time** — update the cron schedule
- **Change classification rules** — update what counts as "critical"

## Technical Implementation

### Prerequisites
1. **Google Cloud Project** with Gmail API and Google Calendar API enabled
2. **OAuth2 credentials** — `credentials.json` stored at `scripts/credentials.json`
   - Create at https://console.cloud.google.com/apis/credentials
   - Application type: Desktop app
   - Download and save as `credentials.json`
3. **Resend API key** — stored in environment variable `RESEND_API_KEY`
   - Sign up at https://resend.com
   - Verify sender domain or use onboarding@resend.dev for testing
4. **Python 3.9+** with required packages:
   ```bash
   pip3 install google-auth google-auth-oauthlib google-api-python-client requests
   ```

### First-Time Setup
Run the auth script to complete OAuth2 authorization:
```bash
cd .claude/skills/email-summary/scripts
python3 setup_auth.py
```
This opens a browser for Google sign-in and saves `token.json` locally. The token auto-refreshes; you only need to do this once.

### How It Works
1. Cron trigger fires at 5 AM ET -> launches this skill via Claude Code
2. `scripts/gmail_client.py` authenticates with Gmail API via OAuth2
3. Fetches all emails from the previous day (midnight to midnight ET)
4. Claude classifies each email as critical or non-critical based on the rules above
5. `scripts/gmail_client.py` also fetches today's calendar events via Google Calendar API
6. Content is compiled into the HTML email template
7. `scripts/send_summary.py` sends the email via Resend API
8. `scripts/gmail_client.py` marks all non-critical emails as read

### Running Manually
```bash
# Generate and send immediately
cd .claude/skills/email-summary/scripts
python3 send_summary.py

# Or invoke via Claude Code / Dispatch
# "Send me my email summary for today"
```

## Execution Flow (Step by Step)

When this skill is triggered (by cron or manually):

1. **Authenticate** — Load Gmail and Calendar OAuth2 credentials from `scripts/token.json`.
2. **Fetch emails** — Call `scripts/gmail_client.py fetch-emails` to get all emails from the previous day.
3. **Fetch calendar** — Call `scripts/gmail_client.py fetch-calendar` to get today's events.
4. **Classify** — For each email, determine if it's "critical" based on the classification rules. This is done by Claude analyzing the sender, subject, and snippet of each email.
5. **Compose** — Build the summary email content:
   - Write the 3-5 sentence "All Emails" paragraph
   - Build the "Critical Emails" bullet list
   - Build the "Key Meetings" bullet list (or "No meetings" placeholder)
6. **Format** — Inject content into the HTML template.
7. **Send** — Execute `scripts/send_summary.py` with the rendered HTML.
8. **Mark as read** — Call `scripts/gmail_client.py mark-read` with the message IDs of non-critical emails.
9. **Self-anneal** — If any step fails, fix the skill files and retry.

## Classification Examples

**Critical:**
- Email from "Chase Bank" re: "Your statement is ready" -> Critical (financial)
- Email from "John Smith" re: "Quick question about Saturday" -> Critical (personalized, action needed)
- Email from "DocuSign" re: "Please sign: Lease Agreement" -> Critical (action required)

**Not Critical:**
- Email from "Nike" re: "Spring Sale - 30% Off" -> Not critical (marketing)
- Email from "Medium Daily Digest" re: "Today's top stories" -> Not critical (newsletter)
- Email from "GitHub" re: "Your weekly digest" -> Not critical (automated digest)
